Names the storyteller invents on the spot (Portal's engines Jiraki/Jameis/Jaraqui, Pataki, Bibi, Foxy, Squishy Baby; Cruel Baby's Khadija/Longer/Yummy). No external truth -- the only defect is inconsistency: the same made-up name spelled several ways. Pataki is a clean example: Whisper also rendered it 'Bacchus', so Pataki/Bacchus is one improvised name with two spellings -- exactly this case. Detect by clustering phonetically-similar name tokens within a story and flagging any cluster with more than one spelling (an LLM does this well; phonetic clustering is the cheaper code version). Per the project owner: this must run AFTER the session is split into stories, because a name can differ legitimately across stories -- so story-splitting is a prerequisite. Detection only flags the inconsistency; choosing the winning spelling is a separate fix decision. (Caveat: 'longer'/'yummy'/'james' are also common words, so their counts are soft.)

* Mode 3 — missed real speech. A coder can only flag a miss when something shows up to click — a half-dropped line, or a visible gap. But often someone speaks and the transcript shows nothing at all for that moment — no line, not even an “[unintelligible]” marker — so there is nothing to click and it can’t be hand-counted. A separate script catches these by comparing who was talking (from the speaker detector) against what got written down. It found 39 clear cases (Portal 29, Moon 4, Cruel Baby 4, Rubber Ducky 2, Pandavas 0); the Total adds those to the 18 a coder could see.

** Mode 7 — word at the wrong time. Whisper guesses each word’s start and end time instead of measuring it, so the times drift — usually the first word of a segment, anchored to the silence just before anyone speaks. It’s hard to spot by eye, so the hand count (17) is far too low. Checking every word against the actual sound, a script found 96 words clearly sitting on silence with their real audio right beside them. Treat that as a rough floor, not the true size: measured as a spread, about 12% of all words start ≥0.1s late, 8% ≥a quarter-second, 3% ≥half a second — over a thousand words. The fix is “forced alignment” (re-snapping each word to the audio), tracked as TMAS-50.
